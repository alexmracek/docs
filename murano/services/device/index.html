Device Gateway Service

API allowing communication with IoT devices

Operations

  • Device.productInfo() - get product information
  • Device.write() - write data to a device
  • Device.rpcCall() - proxies One Platform RPC API
  • Device.deviceCallback() - One Platform service call callback from Murano

Events

  • datapoint - incoming data
  • service_call - service call data

Operations

productInfo

get product information

Arguments
parameters - object - Object containing service call parameters.
Name Type Description
pid string Product ID
Responses
Product information object - object
Name Type Description
id string Identifier of the product.
rid string Resource Identifier (RID) of the product.
label string Display name of the product.
endpoint string URL of the device data endpoint.
Example
local parameters = { 
 pid = "minim"
 }

response = Device.productInfo(parameters)

write

write data to a device

Arguments
parameters - object - Object containing service call parameters.
Name Type Description
pid string Product id
device_sn string Device identity
Example
return Device.write({
  pid="xxxxxxxxxxxxx",
  ["device_sn"]="1",
  state="1"
})

rpcCall

Convert the Product ID ({pid}) to its CIK, add the CIK as the rpc auth object, and forward the request.

See http://docs.exosite.com/rpc/ for argument and response details.

Arguments
parameters - object - Object containing service call parameters.
Name Type Description
pid string Product ID, need to be found in product ID list of solution service config for access rights.
auth object -
auth.client_id string Client context
calls [ object ] -
calls[].id string Call response mapping ID
calls[].arguments [ string ] -
calls[].procedure string -
Responses
RPC call Successful - [ object ]
Example
local parameters = { 
 pid = "ad", 
 calls = { 
 { 
procedure = "dolore anim fugiat ullamco", 
arguments = { 
 "sit eu voluptate ut consectetur"
 }
}, 
 { 
procedure = "id ad commodo amet", 
arguments = { 
 "incididunt mollit aliqua ut", 
 "non aliqua esse fugiat", 
 "magna Ut occaecat in minim", 
 "ni", 
 "non"
 }
}, 
 { 
procedure = "Ut qui Lorem", 
arguments = { 
 "eu do", 
 "elit Duis adipisicing ut deserunt", 
 "laboris Duis", 
 "velit incididunt officia deserunt"
 }, 
id = "eu"
}, 
 { 
procedure = "ad", 
arguments = { 
 "officia in dolor id minim", 
 "adipisicing non proident", 
 "esse dolore aliquip Ut in", 
 "in", 
 "nulla enim commodo"
 }
}
  }, 
 auth = { }
 }

response = Device.rpcCall(parameters)

deviceCallback

One Platform service call callback from Murano

Arguments
parameters - object - Object containing service call parameters.
Name Type Description
code integer The http response status code
headers object Optional http response headers
message string The http response body
request_id string The request ID associated with the ServiceCall
Responses
Callback sent - nil
Invalid request ID - nil
Conflict - nil
Unsupported Media Type - nil
Example
local parameters = { 
 code = 43386763, 
 headers = { }, 
 message = "cillum n", 
 ["request_id"] = "consectetur"
 }

response = Device.deviceCallback(parameters)

Events

datapoint

data generated by devices and pushed by the One Platform device API

Arguments
data - object - The data from the device
Name Type Description
api "write", "record" Provider API
pid string Device product ID
rid string Unique device resource ID
seq integer The message sequence number for specific resource ID
alias string Device resource alias
value [ integer
or number
or string ] {..2}		 Data transmitted by the device.
If api == write, the first value is 'live',
If api == record, the first value is the event timestamp as integer.
The second value is always the data transmitted, either number or string.
device_sn string Device Serial number
source_ip string The device source ip
timestamp integer Server recording time for the event
Example
function handle_device_datapoint (data)

 ... 

end
